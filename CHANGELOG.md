## [0.3.6] — 2026-05-04

### Змінено
- **Dev-збірка ізольована від release** — новий `src-tauri/tauri.dev.conf.json` overlay задає `identifier = com.signalist.app.dev` і `productName = "Signalist Dev"`. Dev і release більше не діляться `settings.json`, WebKit data stores (сесії Telegram/WhatsApp) та іншими даними застосунку. Команда запуску: `npm run tauri:dev`

## [0.3.5] — 2026-05-04

### Додано
- **Теми сайдбару** — чотири режими: `SYS` (слідує macOS appearance), `LITE` (завжди світла), `DARK` (завжди темна), `AUTO` (автодетекція фону активного месенджера/шортката). Перемикач — кнопка у панелі налаштувань під шестернею, режим зберігається між сесіями
- **Світла тема** — тепла кремова палітра (`#f2efe8`) з читабельним лісово-зеленим акцентом (`#3a7a3a`)
- **Авто-детекція теми** — inject-скрипти (Telegram, WhatsApp, custom shortcuts) вимірюють luminance фону сторінки і повідомляють сайдбар при активації webview та зміні теми всередині месенджера

### Змінено
- **Ширина сайдбару: 72 → 64 px**
- **macOS Vibrancy** — сайдбар відображає розмитий фон того, що знаходиться за вікном (`NSVisualEffectView`, Sidebar material)
- **Кольорова палітра** — теплий вугільний (`#1e1b18`) замість Catppuccin-синього; зелений акцент тепер читається виразніше

### Виправлено
- **Витік GPU-сурфейсів від custom shortcuts і user messengers** — не-активні shortcut-webview стискаються до 0×0, що змушує WKWebView звільнити IOSurface backing store; розмір відновлюється при наступному показі
- **Бурст OS-ниток у дебаунсі сповіщень** — при швидких змінах лічильника непрочитаних тепер на месенджер створюється максимум одна нитка замість окремої на кожну подію
- **Дублікати MutationObserver у WhatsApp інжекті** — `setupObservers()` тепер прикріплює observer лише до першого знайденого контейнера

## [0.3.4] — 2026-04-30

### Виправлено
- **Auto-update нарешті працює** — пакет `latest.json` тепер коректно генерується і публікується разом із релізом. До цього updater-артефакти `.app.tar.gz` і `.sig` зовсім не створювались, бо в `tauri.conf.json` був відсутній `bundle.createUpdaterArtifacts: true` — у Tauri 2.x updater-збірка стала opt-in
- Pipeline збирає реліз одразу як публічний (`releaseDraft: false`), інакше `tauri-action` пропускає крок генерації `latest.json`

### Змінено
- CI публікує **лише macOS arm64** (`aarch64-apple-darwin`); попередні цілі universal/Linux/Windows прибрані. Universal-збірка викликала баг у `tauri-action`, який відкидав `.tar.gz`/`.sig` через невідповідність імен; arm64-only усуває цю гілку
- GitHub Actions оновлено: `actions/checkout@v5`, `actions/setup-node@v5`, Node 22 LTS — прибрано попередження про застарілий Node 20

### Примітка
- Установка з v0.2.0 не може автоматично оновитись до 0.3.4: ключ підпису було змінено в гілці `0.3.x`, і v0.2.0 перевіряє підпис старим pubkey. Потрібно один раз встановити `Signalist_0.3.4_aarch64.dmg` вручну — далі авто-оновлення працюватиме
- При першому запуску `.dmg` macOS показує «Програму пошкоджено». Це quarantine-атрибут від ненотаризованої збірки. Виправити: `xattr -cr /Applications/Signalist.app`

## [0.3.0] — 2026-04-29

### Додано
- **Каталог месенджерів** — нова кнопка "+" у бічній панелі відкриває вікно вибору сервісу з готового списку: Instagram, Facebook, Discord, Slack, Signal, LinkedIn, X (Twitter)
- Додані сервіси зберігаються між сесіями у `settings.json` і відображаються в окремій зоні сайдбару між вбудованими месенджерами та шорткатами
- Вже додані сервіси у каталозі відображаються як неактивні (з галочкою) — повторно додати неможливо
- Видалення кастомного месенджера через hover-кнопку "×" (аналогічно шорткатам)
- SVG-іконки для всіх сервісів каталогу

### Змінено
- Середня зона бічної панелі (месенджери + шорткати) тепер прокручується при переповненні
- Scrollbar у бічній панелі прихований; зверху та знизу прокрутної зони — плавне градієнтне затухання через `mask-image`

## [0.2.0] — 2026-04-12

### Додано
- Новий логотип і оновлені іконки застосунку (трей, Dock, інсталятор)

### Змінено
- Рефакторинг Rust-бекенду: структура команд, управління стейтом

### Виправлено

- Логіка debounce сповіщень переписана через generation counter — виключає race condition при швидких змінах
- Хоткей Show/Hide не спрацьовував коректно у повноекранному режимі

## [0.1.0] — 2026-03-20

### Перший публічний реліз

- **Telegram і WhatsApp Web** — вбудовані як дочірні webview з ізольованими data store
- **Бічна панель 72 px** — glassmorphism UI на Tailwind v4, Catppuccin-палітра
- **Бейджи непрочитаних** — JS-ін'єкція читає лічильник з DOM/заголовка, передає через Tauri IPC
- **Системні сповіщення** — push при нових повідомленнях, cooldown 5 с
- **Глобальний хоткей** — Show/Hide вікна (`⌘⇧S` за замовчуванням, налаштовується)
- **Кастомні веб-шорткати** — додайте будь-який сайт з іконкою або ініціалом
- **Автозапуск при вході** в систему
- **Трей-меню** — Show/Hide, перемикач Dock, Quit; іконка стає яскравою при непрочитаних
- **Приховування з Dock** — режим menu-bar-only
- **Автоматичне оновлення** — перевірка GitHub Releases при кожному старті, встановлення в один клік
